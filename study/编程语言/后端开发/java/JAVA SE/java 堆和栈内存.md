这里介绍的是 JVM 内存中的堆和栈内存。

#### 堆内存和栈内存比较

```
1. 作用比较
	堆内存：
        用于存储java中的对象。
        无论是成员变量、局部变量、类变量它们指向的对象都存储在堆内存中。
        堆内存的对象对所有线程都是可见的，堆内存中的对象可以被所有线程访问。
        堆内存中的对象对所有的线程课件，堆内存中的对象可以被所有线程访问。
     栈内存：
     	用于存储局部变量和方法调用。
        栈内存归属于单个线程，每个线程都会有一个栈内存，其存储的变量只能在其所属线程中可见。
        栈内存可以理解为线程的私有内存。
2. 异常错误
	堆内存没有可用的空间存储生成的对象，JVM会抛出java.lang.OutOfMenmeoryError.
	栈内存没有可用的空间存储 方法调用 和 局部变量，JVM会抛出java.lang.StackOverFlowError
3. 空间大小
	栈的内存要远远小于堆内存，如果你使用递归的话，你的栈内存很快就会充满。
	如果递归没有及时跳出，很可能发生StackOverFlowError问题、
	
```
